# Настройка проекта через .env

## Зачем нужен `.env`
Файл `.env` хранит секретные данные (токен бота, ID администратора, пароль) отдельно от кода.  
Это повышает безопасность и не позволяет случайно опубликовать секреты в Git.

## Пошаговая настройка
1. Скопируйте шаблон:
   - Windows (cmd): `copy .env.example .env`
2. Откройте `.env` и заполните реальные значения:
   - `BOT_TOKEN`
   - `ADMIN_ID`
   - `ADMIN_PASSWORD`
   - при необходимости `DATABASE_PATH` и `DEBUG_MODE`
3. Сохраните файл `.env`.

## Где взять значения
- `BOT_TOKEN`: получите у [@BotFather](https://t.me/BotFather) командой `/newbot`.
- `ADMIN_ID`: узнайте через [@userinfobot](https://t.me/userinfobot).

## Важно
- Никогда не отправляйте `.env` другим людям.
- Никогда не добавляйте `.env` в Git-коммиты.
- Перед публикацией репозитория проверьте, что в нём нет реальных секретов.

## Как проверить, что всё работает
1. Убедитесь, что зависимости установлены: `pip install -r requirements.txt`
2. Запустите бота: `python main.py`
3. Если `BOT_TOKEN` или `ADMIN_ID` не заполнены, бот завершится с понятной ошибкой на русском.
4. Если бот запустился без ошибок, конфигурация настроена корректно.
